What This Tool Does And What This Report Communicates

This plugin turns your KiCad design into a reliability model. It reads the component set, applies the IEC TR 62380 failure-rate model to each part, combines those part-level rates into a system result, and then adds uncertainty and sensitivity views so you can see what is solid, what is assumption-driven, and what is worth fixing first.

In plain terms, this report tells you four things. First, the current estimate: system FIT, mission reliability, and MTTF. Second, how much that estimate moves when uncertain inputs such as temperature, duty, or cycling are allowed to vary. Third, which sheets, components, and parameters are doing most of the damage. Fourth, where design effort is most likely to buy back margin if the result is not where you want it yet.

It is best used as a design-review document. The summary gives the headline result, the analysis pages explain what is driving it, and the action pages help you decide what to change next. For the math, assumptions, and interpretation limits behind the numbers, see docs/METHODOLOGY.md.

Methodology & Mathematical Foundations

The calculation flow in this tool is straightforward: estimate each component failure rate with IEC TR 62380:2004, combine those rates into a system result that matches the modelled architecture, then run focused analysis views to show uncertainty, leverage points, and likely paths to improvement. The goal is not to hide the math. It is to make the assumptions visible enough that an engineering team can challenge them, defend them, and act on them.

1. Core Reliability Model

Each component's failure rate λi is computed with the IEC TR 62380 formulation, which captures die-level stress, package-related effects, and overstress terms:

  λi = πdie · λbase + πpkg · λtherm + πEOS · λoverstress

where πdie includes the Arrhenius temperature acceleration (exponential dependence on junction temperature Tj), πpkg includes the Coffin–Manson thermal cycling fatigue (power-law dependence on ΔT and Ncycles), and πEOS models electrical overstress contributions.

The system failure rate comes from the assembled block model. For a simple series path it reduces to the sum of all contributing component failure rates:

  λsys = Σ λi     R(t) = exp(−λsys · t)

2. Tornado Sensitivity (OAT)

The One-At-a-Time (OAT) view perturbs each parameter by a user-specified amount while holding all other parameters at their nominal values. For each perturbation Δx of parameter x:

  FITswing = |λsys(x + Δx) − λsys(x − Δx)| × 109

This is a local deterministic ranking tool. Each bar shows how much system FIT moves when one parameter is perturbed around the current design point and the full IEC formulas are re-evaluated. It is useful for design prioritisation, but it should not be interpreted as a global Sobol analysis.

3. Component Criticality (Elasticity)

The normalized elasticity (log-log derivative) measures the proportional sensitivity of λi to each parameter θ:

  e = (dλ/λ) / (dθ/θ) ≈ (λ(θ+Δθ) − λ(θ−Δθ)) / (2Δθ/θ) / λnom

This central finite-difference approximation is second-order accurate: error is O(h²). Reference: Borgonovo & Plischke (2016) “Sensitivity analysis: A review of recent advances.”

4. Monte Carlo Uncertainty Analysis

The uncertainty analysis pushes plausible input variation through the full model. For each simulation, the tool samples the uncertain inputs, rebuilds the component failure rates, and records the resulting system FIT and mission reliability.

  1. Draw shared parameters once (environmental) and independent parameters per component
  2. Recompute λi through the full IEC TR 62380 formula
  3. Sum to obtain λsys and compute R(t) = exp(−λsys · t)

Convergence rate: O(1/√N) by the Central Limit Theorem. Confidence intervals are computed directly from empirical quantiles of the sample distribution.

PERT distribution (Beta-PERT) with γ=4 is used for expert-elicited bounds. Reference: Vose (2008) “Risk Analysis: A Quantitative Guide.”

5. SRRC (Parameter Importance)

Standardized Rank Regression Coefficients provide a Monte Carlo-based importance ranking for monotonic input-output relationships. In this plugin, SRRC and SRRC² are used as relative screening metrics rather than as a formal substitute for a validated Sobol workflow. Reference: Helton & Davis (2003) “Latin Hypercube Sampling and Propagation of Uncertainty.”

6. Jensen's Inequality Diagnostic

Since R(t) = exp(−λt) is convex in λ, Jensen's inequality gives: E[R(t)] ≥ R(E[λ],t). The report uses this relationship as a consistency check on the Monte Carlo output by comparing the sample mean reliability with the reliability evaluated at the mean sampled failure rate.

Glossary

FITFailure In Time
Unit of failure rate: 1 FIT = 1 failure per 109 device-hours. Widely used in reliability engineering per IEC 61709 and IEC TR 62380.
R(t)Reliability Function
Probability that a system survives to time t without failure. For constant failure rate: R(t) = exp(−λ × t).
MTTFMean Time To Failure
Expected operating time before the first failure: MTTF = 1/λ. For space systems, typically expressed in years.
λFailure Rate
Instantaneous failure rate in failures per hour. Related to FIT by λ = FIT × 10−9.
CIConfidence Interval
Range of values that contains the true parameter with a given probability. A 90% CI means 90% of Monte Carlo samples fall within the bounds.
Monte CarloMonte Carlo Simulation
Statistical technique that propagates parameter uncertainty through repeated random sampling. Each sample draws parameters from their uncertainty distributions and recomputes the system failure rate.
PERTProgram Evaluation and Review Technique
Three-point estimation distribution (min, mode, max) using a scaled Beta distribution. More peaked than Uniform, suitable for expert-elicited uncertainty bounds.
OATOne-At-a-Time Sensitivity
Deterministic method that perturbs one parameter at a time while holding others at their nominal values. Used in Tornado charts.
Tornado ChartTornado Sensitivity Chart
Horizontal bar chart showing the system FIT swing caused by each parameter's perturbation, ordered by impact magnitude.
SRRCStandardized Rank Regression Coefficient
Sensitivity measure from Monte Carlo: the standardized regression coefficient of the rank-transformed output on rank-transformed inputs. SRRC and SRRC² are used here as relative importance indicators for monotonic screening.
ElasticityNormalized Sensitivity
Dimensionless measure: (dY/Y) / (dX/X). An elasticity of 0.5 means a 1% change in parameter X causes a 0.5% change in output Y.
ArrheniusArrhenius Acceleration
Temperature-dependent failure acceleration: πT = exp(Ea/k × (1/Tref − 1/T)). Higher junction temperature exponentially increases failure rate.
Coffin-MansonCoffin-Manson Fatigue
Thermal cycling fatigue model: failure rate contribution proportional to N0.76 × ΔT0.68. Drives package/solder stress.
DeratingElectrical Derating
Operating components below their rated limits (voltage, power, current) to reduce stress and improve reliability. A standard practice in high-reliability design.
Jensen's InequalityJensen Diagnostic
For the convex function R(t) = exp(−λt), Jensen's inequality gives E[R(t)] ≥ R(E[λ],t). The report uses this as a consistency check against the Monte Carlo sample mean, not as a claim about the nominal design point.
